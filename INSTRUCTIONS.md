## 1. Получение API ключей

### 1.1 DeepSeek API

DeepSeek — это LLM (языковая модель), которая анализирует ваши инструкции на естественном языке и преобразует их в структурированные команды для биржи.

**Как получить ключ:**

1. Перейдите на сайт [platform.deepseek.com](https://platform.deepseek.com)
2. Зарегистрируйтесь (требуется email и подтверждение)
3. После входа перейдите в раздел **"API Keys"** (ключи API)
4. Нажмите **"Create API key"**
5. Скопируйте ключ (он показывается **только один раз**!)
6. Пополните баланс аккаунта (DeepSeek — платный API, ~$0.14 за 1M токенов)

**Формат ключа:** `sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

> 💡 **Совет:** DeepSeek — одна из самых дешёвых LLM. Одного запроса хватает на ~50-100 торговых команд. Пополнение на $1 покрывает ~1000 команд.

---

### 1.2 CoinEx API

CoinEx — криптовалютная биржа, на которой будут исполняться ордера.

**Как получить ключи:**

1. Перейдите на сайт [coinex.com](https://www.coinex.com)
2. Зарегистрируйтесь и пройдите верификацию (KYC)
3. Войдите в аккаунт
4. Перейдите в **"Security"** → **"API Management"** (или **"API Keys"**)
5. Нажмите **"Create API Key"**
6. Настройте разрешения:
   
   | Разрешение | Нужно? | Почему |
   |-----------|--------|--------|
   | **Spot Trading** (спот-торговля) | ✅ **Да** | Для создания ордеров |
   | **Spot Order Query** (просмотр ордеров) | ✅ **Да** | Для проверки статуса |
   | **Asset Query** (просмотр баланса) | ✅ **Да** | Для проверки баланса |
   | **Futures** (фьючерсы) | ❌ **Нет** | Безопасность — только спот |
   | **Withdraw** (вывод средств) | ❌ **Нет** | Безопасность |
   
7. Установите **IP-ограничение** (рекомендуется):
   - Если у вас статический IP — укажите его
   - Если динамический — оставьте пустым (менее безопасно)
   
8. Нажмите **"Create"**

**Вы получите два ключа:**
- **Access ID (API Key):** `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX`
- **Secret Key:** `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

> 🚨 **ВАЖНО:** Secret Key показывается только один раз! Сохраните его в надёжном месте (менеджер паролей).

**Комиссии CoinEx (актуально на 2025):**
- Спот-торговля: **0.2%** за сделку (мейкер/тейкер)
- Для VIP-клиентов: от 0.12% (чем больше объём, тем ниже)

## 2. Способы передачи ключей

### Способ 1: Аргументы командной строки (простой, но небезопасный)

```bash
./crypto_trader \
    --deepseek-key "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
    --key "XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX" \
    --secret "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

> ⚠️ **Недостатки:** ключи видны в истории shell (`.bash_history`) и в списке процессов (`ps aux`).

### Способ 2: Переменные окружения (рекомендуется)

```bash
# Устанавливаем ключи
export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
export COINEX_API_KEY="XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX"
export COINEX_API_SECRET="xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# Запускаем (ключи подхватятся автоматически)
./crypto_trader
```

Чтобы добавить в `~/.bashrc` (автозагрузка при входе):
```bash
echo 'export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"' >> ~/.bashrc
echo 'export COINEX_API_KEY="XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX"' >> ~/.bashrc
echo 'export COINEX_API_SECRET="xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"' >> ~/.bashrc
source ~/.bashrc
```

> ✅ **Преимущества:** ключи не видны в истории и списке процессов.

### Способ 3: Файл конфигурации (удобно)

Создайте файл `config.json` в папке с программой:

```json
{
    "deepseek_key": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "coinex_key": "XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX",
    "coinex_secret": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "mode": "dry_run",
    "limits": {
        "max_order_usd": 500,
        "daily_loss_limit": 100,
        "max_open_positions": 3
    }
}
```

Затем запускайте:
```bash
./crypto_trader --config config.json
```

> ✅ **Преимущества:** можно хранить несколько конфигов для разных стратегий.

### Способ 4: Файлы ключей (разработка)

Программа поддерживает хранение ключей в отдельных файлах с ограниченным доступом:

```bash
# Сохранить ключи (автоматически выставляются права 600)
echo "sk-xxxxxxxx" > .deepseek_key && chmod 600 .deepseek_key
echo "XXXXXXXX-XXXX" > .coinex_key && chmod 600 .coinex_key
echo "xxxxxxxxxxxx" > .coinex_secret && chmod 600 .coinex_secret
```

> ⚠️ **Внимание:** эта функция пока не реализована (будет добавлена в ближайшее время).

## 4. Режимы работы

### ⚪ DRY RUN (по умолчанию) — Безопасный режим

```bash
./crypto_trader                               # явно не указан режим
./crypto_trader --dry-run                      # явно указан dry-run
```

**Что происходит:**
1. ✅ Команда отправляется в DeepSeek
2. ✅ DeepSeek возвращает структурированный JSON
3. ✅ Валидатор проверяет все риски
4. ✅ **Ордер НЕ отправляется на биржу**
5. ✅ Выводится предпросмотр: детали, комиссия, тейк/стоп
6. ✅ Команда логируется в SQLite + JSONL

**Для чего:** Тестирование стратегий, проверка работы LLM, отладка.

---

### 🔴 LIVE — Реальный режим

```bash
./crypto_trader --live
```

**Что происходит:**
1. ✅ Запрос подтверждения: "Подтвердите переход в LIVE режим (yes/no):"
2. ✅ Команда отправляется в DeepSeek
3. ✅ Валидатор проверяет риски
4. ✅ Запрос подтверждения перед каждым ордером
5. ✅ **Ордер отправляется на CoinEx**
6. ✅ Результат логируется

**Предостережения:**
- 🚨 **Используются реальные средства!**
- 🚨 Каждый ордер требует явного подтверждения
- 🚨 Рекомендуется сначала настроить лимиты:

```bash
# Внутри программы:
> лимиты
  Максимальный ордер: 1000 USDT
  Дневной лимит убытка: 200 USDT

# Изменить лимиты пока можно только через SQLite:
sqlite3 crypto_trader.db
UPDATE config SET value = '50' WHERE key = 'max_order_usd';
.quit
```

## 5. Решение проблем

### ❌ "API ключ DeepSeek не установлен"

```bash
# Проверьте, установлена ли переменная окружения
echo $DEEPSEEK_API_KEY

# Если пусто — установите
export DEEPSEEK_API_KEY="sk-ваш_ключ"
```

### ❌ "Ошибка HTTP запроса"

```bash
# Проверьте интернет-соединение
ping -c 3 api.deepseek.com
ping -c 3 api.coinex.com

# Проверьте, не блокирует ли фаервол
curl -I https://api.deepseek.com/v1/chat/completions
```

### ❌ "Ответ LLM не соответствует JSON-схеме"

```bash
# Попробуйте переформулировать команду:
# Плохо: "сделай что-нибудь"
# Хорошо: "купи 0.1 ETH по рынку"
# Хорошо: "продай 0.5 BTC по лимиту 65000"
```

### ❌ "Недостаточно USDT"

```bash
# Проверьте баланс
> баланс

# В симуляции баланс $10,000 USDT — этого достаточно для тестов
# В реальном режиме пополните счёт на бирже
```

### ❌ DeepSeek возвращает неверные данные

```bash
# DeepSeek редко ошибается при правильном промпте.
# Проверьте, что вы используете:
# - Чёткие формулировки (купи/продай, количество, пара)
# - Поддерживаемые пары (BTC/USDT, ETH/USDT, SOL/USDT)
# - Не используйте сленг ("зашортить", "лонгануть")
```

## 6. Безопасность

### 🔐 Хранение ключей

| Метод | Безопасность | Удобство |
|-------|-------------|----------|
| Аргументы CLI | ❌ Низкая | ✅ Высокое |
| Переменные окружения | ✅ Средняя | ✅ Высокое |
| Файл конфигурации (chmod 600) | ✅ Средняя | ✅ Высокое |
| OS Keyring (в разработке) | 🏆 Высокая | Среднее |

### 🛡️ Защитные механизмы программы

1. **DRY RUN по умолчанию** — случайно не наторгуете
2. **Подтверждение LIVE** — явное согласие на реальную торговлю
3. **Лимиты** — максимальный ордер, дневной убыток
4. **Circuit breaker** — 3 ошибки → 5 мин паузы
5. **Только спот** — фьючерсы и маржа заблокированы в коде
6. **Валидация цен** — отклонение >10% блокируется
7. **Аудит** — каждое действие логируется в SQLite + JSONL

### 📝 Чек-лист перед LIVE-торговлей

- [ ] Установлены лимиты (начните с `max_order_usd: 10`)
- [ ] Проверен баланс на бирже
- [ ] Протестировано в DRY RUN (несколько команд)
- [ ] DeepSeek корректно распознаёт команды
- [ ] Рядом есть кнопка выключения (Ctrl+C)
